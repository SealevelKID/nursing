# 💡 專案初心

本專案的誕生，是為了「讓政府公開資訊更便民」。
臺灣長照機構的官方原始資料往往分散、格式多變，且常因機構改名或更換負責人而產生資訊斷層。我們希望透過輕量、直覺的數位工具，協助家屬在為長輩做關鍵規劃時，能一鍵橫跨行政裁罰、司法起訴與全網新聞。

## 📂 檔案架構與作用介紹
## 🌐 前端網頁端

nursingindex.html
主網頁入口。建立乾淨、無廣告的卡片式佈局，並在頁尾貼心加入符合人性、不嚴肅的溫馨提示警語。

nursingstyle.css
網頁外觀外衣。專為「電腦版」與「手機版」雙平台進行文字大小與行高微調，確保家屬在戶外用手機查閱時依然清晰易讀。

nursingscript.js
核心前端大腦。負責動態渲染機構卡片，並搭載「智慧防金蟬脫殼」邏輯——自動掃描備註欄位中的改名軌跡，將新舊店名、負責人、首支電話同步送出，交織成密不透風的 Google 查詢網。

## ⚙️ 後端與自動化

data.json
結構化資料庫。存放清洗完畢後的長照機構基礎欄位，供前端即時調用，免去動態資料庫的沉重負擔。

process.py
後端自動化腳本。負責讀取並清洗 Excel 原始數據。現已升級為「變更即時觸發」機制，只要 Excel 檔有更新，腳本便會自動執行清洗並更新 data.json。

.yaml (工作流排程設定)
自動化核心調度。負責監聽 Excel 檔案的變更狀態以實現無人值守更新，並控管定時發送通知的排程時間。

## 🤖 半人工維護與提醒機制

本專案採用「人機協作的半人工維護」架構，整合定時排程：

**年中/年末定時提醒**：於每年年中（6月）與年末（12月），系統將透過鎖定密鑰的 Telegram 機器人，自動向網頁管理者發送更新提示，提醒管理者需前往下載最新官方 Excel 檔案。
